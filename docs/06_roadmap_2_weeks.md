# Roadmap (2 weeks)

Приоритет roadmap обновлен под `docs/14_rental_lifecycle.md`.

## Week 1: доменная модель и backend

- Разделить сущности:
  - `bike` - физический велосипед;
  - `rental` - жизненный цикл велосипеда на главном экране админа;
  - `client_rental` - период пользования велосипедом конкретным клиентом.
- Убрать модель `rental = client + bike` из планов и реализации.
- Обеспечить инвариант: один велосипед может иметь только одну неудаленную `Аренду`.
- Добавить статусы `rental.pipeline_status`:
  - `long_term`;
  - `soon_return`;
  - `in_stock`.
- Реализовать закрытие текущей `client_rental` при переходе в `in_stock`.
- Реализовать создание новой `client_rental` внутри существующей `rental` без создания новой `rental`.
- Перенести логин/пароль на уровень `client_rental`.
- Добавить генерацию пары `логин + пароль` одним действием (кнопка «Сгенерировать» в `Аренде` в статусе `in_stock`) и проверку неповторяемости паролей (см. docs/14_rental_lifecycle.md §4).
- Обеспечить инвариант «у каждой client_rental есть непустые логин и пароль» — включая backfill для legacy-данных при нормализации state.
- Обновить ledger и платежи так, чтобы они ссылались на `client_rental_id`.
- Реализовать финальный расчет долга закрытой `client_rental` строго по дням.
- Реализовать удаление `rental` как редкий сценарий вывода велосипеда из эксплуатации:
  - закрыть активную `client_rental`;
  - перенести остаточный долг на клиента;
  - скрыть карточку аренды с главного экрана.

## Week 1: iOS admin flow

- Главный экран админа должен показывать список `Аренд`, а не список клиентов.
- Цвет рамки аватарки:
  - зеленый - долгосрочная;
  - желтый - скоро вернет;
  - фиолетовый - у меня.
- Карточка `Аренды` в статусе `in_stock`:
  - показывает прочерки вместо статистики;
  - не показывает историю платежей текущей аренды;
  - отключает кнопку `+ Корректировка`;
  - показывает кнопку выбора клиента;
  - позволяет редактировать логин и пароль для нового цикла;
  - копирует логин/пароль только если оба поля заполнены;
  - показывает информационные сообщения при незаполненных полях.
- Кнопка `Создать новую!` должна:
  - валидировать выбранного клиента и новый пароль;
  - создать `client_rental`;
  - перевести текущую `rental` в `long_term`;
  - не создавать новую `rental` с тем же велосипедом.
- Экран выбора клиента:
  - открывается отдельным окном;
  - переиспользует экран списка клиентов;
  - исключает клиентов с активными арендами;
  - сортирует свободных клиентов по алфавиту;
  - поддерживает одиночный выбор с галочкой.

## Week 2: каталоги, клиентская часть и стабилизация

- Добавить нижнюю навигацию админки:
  - главный экран;
  - список клиентов;
  - список велосипедов.
- Доработать карточку клиента:
  - список `client_rental`;
  - активная/завершенная аренда;
  - финальный долг;
  - платежи и корректировки по каждой `client_rental`;
  - клиентская задолженность после удаления `rental`.
- Клиентская часть:
  - вход по логину/паролю конкретной `client_rental`;
  - просмотр активной или завершенной аренды;
  - оплата долга завершенной аренды без новых начислений.
- Обновить YooKassa:
  - платежи привязываются к `client_rental_id`;
  - налоговый режим берется из `client_rental.tax_mode`.
- Добавить backend integration tests:
  - один велосипед нельзя привязать к двум неудаленным `rental`;
  - `in_stock -> Создать новую!` не создает новую `rental`;
  - закрытая `client_rental` не получает новые начисления;
  - частичная оплата закрытого долга уменьшает долг;
  - удаление `rental` переносит долг на клиента.
- Добавить iOS UI tests для:
  - выбора клиента в статусе `in_stock`;
  - генерации/копирования доступа;
  - запуска нового цикла без создания дубликата аренды.

## Done Criteria

- Создание велосипеда предшествует созданию `Аренды`.
- Главный экран админа стабильно отображает число карточек `Аренды` меньше или равное количеству велосипедов (больше не может быть).
- Переход `long_term/soon_return -> in_stock` закрывает текущую `client_rental`.
- Переход `in_stock -> long_term` создает новую `client_rental` внутри той же `rental`.
- Старые логины закрытых `client_rental` продолжают открывать клиентскую часть для просмотра и оплаты долга.
- Удаление `rental` не теряет долг клиента и убирает карточку с главного экрана.
