# Roadmap (2 weeks)

## Week 1
- Рефактор модели данных на 3 сущности:
  - `client` (персональные данные + телефоны).
  - `bike` (фото, модель, стоимость недели, серийные номера).
  - `rental` (связка client+bike, период, документы, комментарии, доступ клиента).
- Зафиксированные правила недели:
  - `1 день = неделя / 7`, `2 недели = неделя * 2`, `месяц = 4 недели`.
  - Округление сумм всегда вверх до десятков рублей.
  - Регистрация клиентом отключена, доступ клиенту назначается админом из формы аренды.
  - Долг и корректировки считаются только через ledger-записи.
- Разделение потоков создания:
  - Сервисный экран: `Новый клиент` и `Новый велосипед`.
  - Отдельная карточка `Новая аренда` (центр рабочей логики админа).
- Перенос полей по карточкам:
  - `Новый клиент`: ФИО, адрес, паспорт, телефоны (динамическое добавление полей).
  - `Новый велосипед`: фото из галереи, модель, недельный тариф, 4 серийных номера.
  - `Новая аренда`: блок `Доступ клиента` (login/password), блок `Документы и комментарии`, выбор клиента и велосипеда из списков.
- Backend/API:
  - Новые endpoints: list/create clients, list/create bikes, create rentals.
  - Rental creation должен уметь назначать/обновлять клиентский доступ (login/password).
  - Валидации и человекочитаемые ошибки без падений UI.
- Хранение и устойчивость:
  - Сохранение всех сущностей в PostgreSQL, загрузка после перезапуска.
  - Защита от частичных ошибок записи (атомарные изменения состояния).
- iOS admin-first реализация:
  - Полная адаптация под новую backend-модель.
  - Одинаковое поведение на simulator и реальном iPhone.
  - Базовые unit/integration тесты для критичных операций создания.

## Week 2
- Карточка клиента + история аренд.
- Редактирование комментариев и ссылок.
- Интеграция YooKassa (create payment + webhooks + idempotency).
- Проводки ledger.
- Debt adjustments.
- Полировка edge-cases по расчетам.
- Тестирование (unit + integration).
- Beta rollout.
- Bugfix и релиз-кандидат.
