# Money and Debt Rules (Frozen)

## 1. Базовая ставка
- Админ задает недельную ставку клиента: `W` (руб).

## 2. Тарифы оплаты
- `day_amount = ceil_to_10(W / 7)`
- `week_amount = W`
- `two_weeks_amount = W * 2`
- `month_amount = W * 4`

Где `ceil_to_10(x)` = округление в большую сторону до десятков рублей.
Пример: `431 -> 440`.

## 3. Модель баланса
В системе ведется ledger (проводки):
- `CHARGE` - начисление аренды
- `PAYMENT` - оплата клиента (ЮKassa/ручная)
- `ADJUSTMENT` - ручная корректировка админа

Поля проводки:
- `amount_rub`
- `direction`:
  - `+1` увеличивает долг
  - `-1` уменьшает долг
- `created_at`
- `comment`

Итоговый долг:
- `debt = max(0, sum(CHARGE) - sum(PAYMENT) - sum(ADJUSTMENT_REDUCING_DEBT) + sum(ADJUSTMENT_INCREASING_DEBT))`

## 4. Правило начисления для MVP
Чтобы совпасть с согласованным примером, в MVP применяем недельный цикл начисления:
- Начисление `W` выполняется каждые 7 дней аренды (пакетно, за новую неделю).
- В момент старта аренды обычно фиксируется первая оплата недели.
- На 8-й день появляется долг за следующую неделю (`W`), если платежа не было.

## 5. Корректировка админом
- Корректировка хранится как отдельная проводка `ADJUSTMENT`.
- Сумма корректировок отображается накопительно.
- Пример:
  - две корректировки `-1000` и `-500`
  - суммарная корректировка = `-1500`

## 6. Формула отображения прибыли
- `profit_total = sum(success_payments)`
- При отсутствии долга в правом блоке списка показываем `profit_total` на зеленом фоне.
- При наличии долга показываем `debt` на красном фоне.

## 7. Комментарий по спорной точке
Дневной тариф (`W/7`) используется для платежной кнопки "1 день".
Основное начисление долга в MVP фиксируем недельными шагами, чтобы поведение совпадало с бизнес-примером.
