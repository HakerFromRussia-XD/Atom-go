# Scope Freeze (MVP v1)

## 1. Роли и вход
- Экран логина/пароля открывается первым.
- Если логин относится к роли `client` -> клиентская часть.
- Если логин относится к роли `admin` -> админская часть.
- Самостоятельной регистрации клиента нет.
- Новых клиентов создает только админ.

## 2. Клиентская часть
- Отображать модель выданного велосипеда.
- Отображать дату начала аренды.
- Отображать дату, до которой аренда оплачена (`paid_until`).
- Отображать сумму долга (если есть).
- Кнопки оплаты:
  - `1 день`
  - `1 неделя`
  - `2 недели`
  - `1 месяц`
  - `оплатить весь долг` (точная сумма)

## 3. Админская часть
- Список всех клиентов с выданными велосипедами.
- В ячейке клиента:
  - аватар велосипеда
  - имя арендатора в заголовке
  - подпись о статусе оплаты: сколько дней оплачено вперед или сколько дней просрочки в подзаголовке
  - справа:
    - красный блок с долгом, если долг > 0
    - зеленый блок с прибылью за все время аренды, если долга нет
- По тапу на сумму справа: ручная корректировка расчётной суммы выплат клиента за весь период аренды. 
- Создание клиента:
  - логин
  - пароль
  - привязка модели велосипеда
  - недельная ставка аренды

## 4. Карточка клиента в админке
- Адрес проживания.
- Телефоны с подписями (типы телефонов настраиваемые).
- Паспортные данные.
- ФИО.
- Общая сумма платежей за все время.
- Текущий долг.
- Список аренд:
  - аватар велосипеда
  - период аренды (дата начала - дата конца)
  - модель велосипеда
  - кнопки:
    - ссылка на видео приема-передачи
    - ссылка на скан договора
    - комментарий (редактируемый)
  - если комментарий уже есть, он отображается под ячейкой аренды

## 5. Нефункциональные фиксации
- Язык MVP: только русский.
- Push-уведомления в MVP не требуются.
- Документы и медиа хранятся как внешние ссылки (Google Drive, YouTube и т.д.).
- Android и iOS приложения должны быть визуально одинаковыми и идентичными по функционалу на каждом этапе разработки.
