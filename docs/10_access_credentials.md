# Тестовые Логины И Пароли (MVP)

Документ с текущими тестовыми учетными данными для входа в приложение.

Источник данных: `backend/app/src/main/kotlin/com/atomgo/backend/infra/InMemoryStore.kt`.

Важно: по новой модели логин и пароль клиента принадлежат не клиенту целиком, а конкретной `клиентской аренде`. Старые логины закрытых `клиентских аренд` должны продолжать работать для просмотра этой аренды и оплаты оставшегося долга.

## Вход в админскую часть

- `login`: `admin`
- `password`: `admin123`
- `role`: `admin`
- `tax_mode`: `self_employed`

- `login`: `admin_ip`
- `password`: `adminip123`
- `role`: `admin`
- `tax_mode`: `individual_entrepreneur`

## Вход в клиентскую часть

- `login`: `client1`
- `password`: `client123`
- `role`: `client`
- привязка: тестовая `client_rental`

- `login`: `client2`
- `password`: `client234`
- `role`: `client`
- привязка: тестовая `client_rental`

## Примечание

- Это тестовые учетные данные из in-memory seed для локальной разработки.
- Для production эти пароли использовать нельзя.
- Новые пары `логин + пароль`, генерируемые админом для `клиентских аренд` через кнопку «Сгенерировать», создаются одновременно (см. docs/14_rental_lifecycle.md §4). Пароль должен быть сложным и не должен повторять ранее созданные пароли в системе.
- Backend должен проверять неповторяемость пароля через безопасный отпечаток, не через хранение пароля открытым текстом.
- Инвариант: у каждой `клиентской аренды` (включая закрытые) обязаны быть непустые логин и пароль.
